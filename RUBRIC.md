# Evaluation Rubric

Rubric used to judge each baseline run and refinement run for the slot machine app. Record assessment in each runs Markdown assessment/observations file.

## Scoring Scale

- 0 = missing or broken
- 1 = weak or incomplete
- 2 = acceptable
- 3 = strong

## App Quality

### 1. Core Slot Machine Behavior
- 3: Spin works correctly, token win/spend is clear, and results update consistently.
- 2: The spin and token counting mainly works, but has small bugs or unclear feedback.
- 1: Some parts work of the machine work, but major parts are missing or inconsistent.
- 0: Does not function as a slot machine.

### 2. AI Satire and Theme
- 3: Theme is clear, funny, and tied to AI/token jokes.
- 2: Theme is present but not very strong.
- 1: Theme appears only a little or feels unclear.
- 0: No clear AI satire/theme.

### 3. User Experience
- 3: Easy to use, clear buttons/messages, and smooth flow/fun experience.
- 2: Mostly usable but may have some minor inconsistencies that prevent a better experience.
- 1: Noticeable UX issues tha make it harder to use.
- 0: Very confusing or broken interface.


## Code Quality

### 1. Vanilla Web Technology Use
- 3: Fully uses plain HTML/CSS/JS and platform APIs correctly.
- 2: Mostly vanilla, with minor unnecessary complexity or bugs.
- 1: Weak use of required stack or mixed approach.
- 0: Does not follow required vanilla web stack.

### 2. Readability and Organization
- 3: Clean structure that is easy to read, and organized logically.
- 2: Understandable but a bit messy in places.
- 1: Hard to follow due to clutter or poor structure.
- 0: Very difficult to read or evaluate.

## Overall Judgment

- Strong: Most criteria score 3, and no major category is below 2.
- Acceptable: Core behavior works, theme is present, and code quality is decent.
- Needs Work: One or more core areas are missing, broken, or difficult to maintain.

## Notes

- Focus first on whether the app works as a slot machine.
- Then check code quality/usage of vanilla HTML, CSS, JavaScript, and platform APIs.
- Finally, judge the AI satire/theme.
